## 1. 项目概述

### 1.1 背景

Pavo AI Agent v1.0 完成了从故事输入到分镜脚本的核心管线。v2.0（MCP Server 层）将系统能力以标准化协议暴露给 AI 客户端。在此基础上，v2.1 的核心目标是解决两个根本性问题：

| 问题 | 现状 | v2.1 目标 |
|------|------|-----------|
| **创作连续性** | 每次创作"从零开始"，无历史记忆 | 跨会话的用户风格/偏好记忆，自动上下文注入 |
| **专业深度** | 分镜缺乏影视专业水准 | RAG 知识库注入专业影视知识，提升分镜质量 |

### 1.2 v2.1 核心目标

| 目标 | 说明 | 衡量标准 |
|------|------|---------|
| 个性化创作记忆 | 系统记住用户的创作风格、常用角色、偏好场景 | 连续3次创作风格一致率 > 80% |
| 影视知识注入 | 分镜生成时自动检索影视专业知识并注入 Prompt | 专业术语正确率 > 90% |
| 透明集成 | Memory/RAG 对 MCP Tool 调用方透明，无需额外参数 | 现有 Tool 调用无需修改 |
| 低延迟 | Memory/RAG 检索总耗时 < 500ms | P95 检索延迟 < 500ms |

### 1.3 范围界定

| 包含 | 不包含 |
|------|--------|
| Agent Memory 存储与检索系统 | 修改现有 Agents 管线执行逻辑 |
| 4 个 Memory MCP Tools | 前端 UI 改造 |
| RAG 知识库构建 Pipeline | 大规模知识库分布式部署 |
| RAG 检索服务与 Agent 注入 | 多模态 RAG（图片/视频检索） |
| Memory Middleware 透明集成 | 用户权限管理系统 |
| pgvector 向量数据库集成 | 外部知识库 API 对接 |
| MCP Tool context 参数预留 | 知识库版权合规自动审核 |
| 种子知识数据（55+ 条高质量知识） | |

## 4. Agent Memory 系统详细设计

### 4.1 记忆层级

```
┌──────────────────────────────────────────────┐
│                Agent Memory                   │
│                                               │
│  Layer 1: 短期记忆 (Session Memory)           │
│  · 存储: Redis (TTL 30 分钟)                  │
│  · 内容: 对话历史, 当前项目上下文, 临时状态     │
│  · 作用域: 单次 MCP 会话                       │
│                                               │
│  ↓ 会话结束 → 重要信息提取                      │
│                                               │
│  Layer 2: 长期记忆 (User Memory)               │
│  · 存储: PostgreSQL + pgvector (1536 维)       │
│  · 生命周期: 永久 (用户可管理)                  │
│  · 内容: 风格偏好, 常用角色, 场景偏好, 创作历史  │
│  · 检索: 语义相似度 (余弦) + 结构化过滤         │
└──────────────────────────────────────────────┘
```

### 4.2 记忆类型

| memory_type | 说明 | 示例 |
|------------|------|------|
| `style` | 用户创作风格偏好 | "偏好温暖治愈风格，色调柔和" |
| `character` | 用户自定义/高频角色 | "小明：5岁男孩，大眼睛，红色T恤" |
| `scene` | 用户偏好/高频场景 | "温馨家庭场景，傍晚暖色灯光" |
| `preference` | 通用偏好设置 | "分镜总时长偏好 45 秒" |
| `story_arc` | 叙事结构偏好 | "偏好三幕剧结构" |
| `feedback` | 用户反馈摘要 | "用户反馈角色细节不够丰富" |

### 4.3 核心接口

```python
# memory/interfaces.py
class MemoryProvider(ABC):
    async def save_memory(self, memory: UserMemory) -> UserMemory
    async def search_memory(self, user_id, query_text, memory_types, limit, min_importance) -> list[UserMemory]
    async def list_memories(self, user_id, memory_type, page, page_size) -> tuple[list, int]
    async def delete_memory(self, user_id, memory_id) -> bool
    async def get_session_context(self, session_id) -> dict
    async def update_session_context(self, session_id, context) -> None
```

### 4.4 数据模型

```python
class UserMemory(BaseModel):
    id: UUID
    user_id: str
    memory_type: str        # style/character/scene/preference/story_arc/feedback
    content: dict           # 记忆内容 (结构化)
    embedding: list[float]  # 向量化表示
    importance: float       # 重要性评分 0-1
    source: str             # auto_extracted / user_saved / feedback_derived
    tags: list[str]
    created_at: datetime
    updated_at: datetime
    accessed_at: datetime

class SessionMemory(BaseModel):
    session_id: str
    user_id: str
    project_id: str | None
    messages: list[dict]    # 对话历史
    context: dict           # 会话上下文摘要
    expires_at: datetime    # TTL

class MemoryQuery(BaseModel):
    user_id: str
    query_text: str
    memory_types: list[str] | None
    limit: int = 5
    min_importance: float = 0.0
    session_id: str | None

## 6. Memory Middleware 设计

### 6.1 设计目标

| 目标 | 指标 |
|------|------|
| 透明集成 | 现有 Tool handler 无需修改 |
| 低开销 | 处理耗时 P95 < 100ms |
| 可降级 | Memory 不可用时不影响核心功能 |

### 6.2 实现方案

```python
# middleware/memory_middleware.py
class MemoryMiddleware:
    """
    MCP Tool 调用拦截中间件。
    在 Tool 被调用前/后自动执行 Memory 操作。
    """

    async def pre_process(self, tool_name: str, arguments: dict) -> dict:
        """
        前置处理：提取 user_id → 查询长期记忆 → 合入 context.memory
        """
        ...

    async def post_process(self, tool_name: str, arguments: dict, result: dict) -> None:
        """
        后置处理：提取本次交互的关键信息 → 存入 Memory Store
        """
        ...
```

### 6.3 集成方式

```python
# main.py (MCP Server 入口)
@server.call_tool()
async def wrapped_call_tool(name: str, arguments: dict):
    augmented_args = await middleware.pre_process(name, arguments)
    result = await original_call_tool(name, augmented_args)
    await middleware.post_process(name, arguments, result)
    return result
```

## 14. 附录：pgvector 快速参考

### 14.1 Docker 配置

```yaml
# docker-compose.yml 修改
postgres:
  image: pgvector/pgvector:pg16    # 替代 postgres:16-alpine
```

### 14.2 向量维度

| 模型 | 维度 | 适用 |
|------|------|------|
| text-embedding-3-small | 1536 | 通用 (推荐) |
| text-embedding-3-large | 3072 | 高精度 |

### 14.3 检索 SQL

```sql
-- 余弦相似度
SELECT id, title, content, 1 - (embedding <=> :query) AS similarity
FROM knowledge_base
WHERE category = ANY(:categories)
ORDER BY embedding <=> :query
LIMIT :top_k;

-- 混合检索 (向量 + 结构化)
SELECT km.*, 1 - (km.embedding <=> :query) AS similarity
FROM user_memories km
WHERE km.user_id = :user_id
  AND km.importance >= :min_importance
ORDER BY km.embedding <=> :query
LIMIT :limit;
```
