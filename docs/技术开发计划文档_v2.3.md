## 1. 项目概述

### 1.1 背景

Pavo AI Agent v1.0 实现了从故事输入到分镜脚本的核心管线。v2.0（MCP Server 层）将系统能力以标准化协议暴露给 AI 客户端。v2.2 作为该系列的最终实施版，在 v2.1 设计方案基础上，吸纳了架构评审的全部反馈，确保 Memory + RAG 系统可落地、可验证、可运维。

| 问题 | 现状 | v2.2 目标 |
|------|------|-----------|
| 创作连续性 | 每次创作"从零开始"，无历史记忆 | 跨会话的用户风格/偏好记忆，自动上下文注入 |
| 专业深度 | 分镜缺乏影视专业水准 | RAG 知识库注入专业影视知识，提升分镜质量 |
| 可运维性 | v2.1 部分设计未细化 | 全链路可观测、可降级、可基准测试 |

### 1.2 核心目标

| 目标 | 说明 | 衡量标准 |
|------|------|---------|
| 个性化创作记忆 | 系统记住用户的创作风格、常用角色、偏好场景 | 连续 3 次创作风格一致率 > 80%；重要性评分有效过滤低价值记忆 |
| 影视知识注入 | 分镜生成时自动检索影视专业知识注入 Prompt | 专业术语正确率 > 90%；RAG 注入目标：分镜智能体（主）+ 角色/场景智能体（辅） |
| 透明集成 | Memory/RAG 对 MCP Client 完全透明 | 现有 Tool 调用无需修改；Middleware 处理 P95 < 100ms |
| 低延迟 | 全链路检索总耗时可控 | Memory 检索 P95 < 500ms；RAG 检索 P95 < 500ms；Embedding 缓存命中率 > 30% |

## 4. Agent Memory 系统详细设计

### 4.1 记忆层级

```
┌──────────────────────────────────────────────────┐
│                  Agent Memory                     │
│                                                  │
│  Layer 1: 短期记忆 (Session Memory)              │
│  · 存储: Redis (TTL 30 分钟，滑动窗口刷新)        │
│  · 内容: 对话历史, 当前项目上下文, 临时状态        │
│  · 作用域: 单次 MCP 会话                          │
│  · 刷新策略: 每次 Tool 调用自动续期 TTL           │
│                                                  │
│  ↓ 会话结束 / 重要事件触发 → 提取 → 评估         │
│                                                  │
│  Layer 2: 长期记忆 (User Memory)                  │
│  · 存储: PostgreSQL + pgvector (1536 维)          │
│  · 生命周期: 永久 (用户可管理 + 低重要性自动清理)   │
│  · 内容: 风格偏好, 常用角色, 场景偏好, 创作历史    │
│  · 检索: 语义相似度 (余弦) + 结构化过滤 + 重要性过滤│
└──────────────────────────────────────────────────┘
```

### 4.2 记忆类型

| memory_type | 来源 | 说明 | 默认 importance |
|------------|------|------|----------------|
| `style` | 自动提取 / 用户保存 | 创作风格偏好 | 0.7 |
| `character` | 自动提取 / 用户保存 | 自定义/高频角色 | 0.8 |
| `scene` | 自动提取 / 用户保存 | 偏好/高频场景 | 0.6 |
| `preference` | 用户保存 | 通用偏好设置 | 0.5 |
| `story_arc` | 自动提取 | 叙事结构偏好 | 0.5 |

### 4.3 重要性评分策略 ★【审核 C1 新增】

```python
# memory/importance.py

class ImportanceStrategy:
    """重要性评分计算策略 —— 决定记忆是否值得留存"""

    @staticmethod
    def from_user_saved() -> float:
        """用户主动保存 → 高重要性 (0.8-1.0)"""
        return 0.9

    @staticmethod
    def from_auto_extracted(mentions: int, recency_days: float) -> float:
        """
        自动提取 → 基于提及次数和时效性

        Args:
            mentions: 同一内容在对话中出现的次数
            recency_days: 距离当前时间的天数 (0-30)

        Formula: min(0.3 + mentions * 0.1 - recency_days * 0.01, 0.8)
        """
        return min(0.3 + mentions * 0.1 - recency_days * 0.01, 0.8)

    @staticmethod
    def from_feedback_derived(rating: str) -> float:
        """用户反馈衍生 → 好评 0.7，差评 0.2"""
        return 0.7 if rating == "up" else 0.2
```

> 低重要性记忆（importance < 0.2）在存储超过 30 天后自动清理。

### 4.4 Embedding 配置 ★【审核 C2 新增】

| 参数 | 值 | 说明 |
|------|-----|------|
| 模型 | text-embedding-3-small | 通用 Embedding，1536 维 |
| 维度 | 1536 | 与 pgvector 索引匹配 |
| Batch 大小 | 100 | 批量处理时每次 100 条 |
| 最大重试 | 3 | 指数退避 |
| 超时 | 30s | 单次 API 调用超时 |
| 缓存 TTL | 3600s (1h) | 相同文本命中缓存时不重复调用 API |

```python
# memory/embedding_client.py

class EmbeddingClient:
    """Embedding 向量化客户端（含缓存）"""

    def __init__(self, model: str = "text-embedding-3-small"):
        self.model = model
        self.dimension = 1536
        self.batch_size = 100
        self.max_retries = 3
        self.timeout = 30.0
        self.cache: dict[str, list[float]] = {}
        self.cache_ttl = 3600  # 1 小时

    async def embed(self, text: str) -> list[float]:
        """单条向量化（带缓存）"""
        cache_key = hashlib.md5(text.encode()).hexdigest()
        if cache_key in self.cache:
            return self.cache[cache_key]
        vector = await self._call_embedding_api([text])
        self.cache[cache_key] = vector[0]
        return vector[0]

    async def embed_batch(self, texts: list[str]) -> list[list[float]]:
        """批量向量化"""
        return await self._call_embedding_api(texts)

    async def _call_embedding_api(self, texts: list[str]) -> list[list[float]]:
        """调用 Agnes AI Embedding API（兼容 OpenAI）"""
        for attempt in range(self.max_retries):
            try:
                # ... API 调用逻辑
                pass
            except Exception:
                if attempt < self.max_retries - 1:
                    await asyncio.sleep(2 ** attempt)
                else:
                    raise
```

### 4.5 Session Memory TTL 刷新策略 ★【审核 C5 新增】

| 机制 | 说明 |
|------|------|
| 初始 TTL | 30 分钟 |
| 刷新条件 | 每次 Middleware 处理 Tool 调用时自动续期 |
| 刷新方式 | 滑动窗口：在 Redis 中重置 TTL 为 30 分钟 |
| 未刷新过期 | 30 分钟内无活动 → 自动删除 |
| 重要事件提取 | 会话即将过期前，自动提取关键信息存入长期记忆 |

```python
# memory/session_store.py
class SessionStore:
    async def touch_session(self, session_id: str) -> None:
        """刷新 Session TTL（滑动窗口）"""
        await self.redis.expire(f"session:{session_id}", 1800)  # 30 分钟

    async def extract_and_promote(self, session_id: str) -> None:
        """会话过期前提取重要信息 → 提升到长期记忆"""
        session = await self.get_session(session_id)
        important_items = self._extract_important_items(session.messages)
        for item in important_items:
            importance = ImportanceStrategy.from_auto_extracted(
                mentions=item.mentions,
                recency_days=0
            )
            if importance >= 0.5:  # 只有足够重要的才提升
                await self.memory_store.save(UserMemory(...))
```

### 5.3 Re-ranker 重排策略 ★【审核 C4 新增】

```python
# rag/reranker.py

class Reranker:
    """多级重排策略 —— 提升检索精准度"""

    async def rerank(
        self,
        candidates: list[KnowledgeEntry],
        query: RAGQuery,
        top_k: int = 5
    ) -> list[KnowledgeEntry]:
        """
        重排策略：
        1. 向量相似度分数 (cosine) —— 基础分
        2. category 匹配度加成 —— 与查询同类别 +0.1
        3. 优先级权重 (priority * 0.05) —— 高优先级前置
        4. 时效性衰减 (旧的 -0.01/月) —— 新知识优先
        """
        scored = []
        for entry in candidates:
            score = self._base_similarity(entry, query)
            if query.categories and entry.category in query.categories:
                score += 0.1
            score += entry.priority * 0.05
            scored.append((score, entry))

        scored.sort(key=lambda x: x[0], reverse=True)
        return [entry for _, entry in scored[:top_k]]

    @staticmethod
    def _base_similarity(entry: KnowledgeEntry, query: RAGQuery) -> float:
        """向量余弦相似度（由 pgvector 返回）"""
        return entry._similarity  # pgvector 查询结果中附带
```

**检索失败降级**：当 RAG 检索返回空或 top-1 相关性 < 0.5 时，降级到原始 Prompt（不注入知识），不报错。

### 6.2 三级降级策略

```python
class MemoryMiddleware:
    async def pre_process(self, tool_name: str, arguments: dict) -> dict:
        try:
            return await self._process_with_memory(tool_name, arguments)
        except MemoryUnavailableError:
            # 级别 1: 服务不可用 → 记录日志，继续执行
            logger.warning("Memory unavailable, continuing without context")
            self._metrics.fallback_total.inc()
            return arguments
        except MemoryTimeoutError:
            # 级别 2: 超时降级 → 使用缓存（如有）
            logger.warning("Memory timeout, using cache if available")
            self._metrics.fallback_total.inc()
            return await self._process_with_cache(tool_name, arguments)
        except Exception as e:
            # 级别 3: 未知错误 → 安全降级，不影响主流程
            logger.error(f"Memory middleware error: {e}")
            self._metrics.fallback_total.inc()
            return arguments
```

```sql
-- 余弦相似度检索（基础）
SELECT id, title, content, category,
       1 - (embedding <=> :query_vector) AS similarity
FROM knowledge_base
WHERE category = ANY(:categories)
ORDER BY embedding <=> :query_vector
LIMIT :top_k;

-- Memory 混合检索（向量 + 结构化）
SELECT km.id, km.content, km.memory_type, km.importance,
       1 - (km.embedding <=> :query_vector) AS similarity
FROM user_memories km
WHERE km.user_id = :user_id
  AND km.importance >= :min_importance
  AND (km.memory_type = ANY(:memory_types) OR :memory_types IS NULL)
ORDER BY km.embedding <=> :query_vector
LIMIT :limit;
```

## 15. Workflow 可视化功能设计 ★ 新增【C15: v2.2 扩展功能】

### 15.1 功能背景

当前 Pavo AI Agent 的 Agent 管线执行状态通过 ChatPanel 中的文本日志展示：

```
[Planner] Analyzing story... ✓
[Character] Generating characters... ✓
...
```

这种方式信息密度低、缺乏直观性。Workflow 可视化功能将 Agent 管线执行过程以可视化流程图形式呈现，让用户直观看到每个 Agent 的执行状态、耗时和产出。

### 15.2 功能目标

| 目标 | 说明 | 优先级 |
|------|------|--------|
| 管线可视化 | 以流程图形式展示 7 个 Agent 的执行顺序和当前状态 | P0 |
| 实时状态更新 | Agent 执行状态变化时前端即时更新（idle/running/completed/failed） | P0 |
| 耗时展示 | 每个 Agent 执行耗时和总耗时预估 | P0 |
| 节点详情 | 点击 Agent 节点可查看输入/输出摘要 | P1 |
| 错误高亮 | 执行失败的 Agent 红色高亮，显示错误信息 | P1 |
| 动画过渡 | Agent 间流转时的平滑动画 | P2 |

### 15.4 技术方案

#### 15.4.1 架构位置

```

                        ├── PipelineGraph (SVG 流程图)

                        ├── AgentDetailPanel (详情弹窗)
                        └── TimelineView (时间线)

#### 15.4.2 后端增强

现有 SSE Stream 的 `agent:progress` 事件需要增加结构化字段：

```

#### 15.4.3 前端组件设计

| 组件 | 职责 | 状态管理 |
|------|------|---------|
| `WorkflowVisualizer` | 整体容器，管理 SSE 连接和状态 | useReducer + context |
| `PipelineGraph` | SVG 管线图布局，节点间连线 | 计算节点位置和连线路径 |
| `AgentNode` | 单个 Agent 节点（状态色、图标、耗时） | props 驱动 |
| `AgentDetailPanel` | 节点详情弹窗 | 受控组件 |
| `TimelineView` | 时间线视图 | 计算时间轴 |

**技术选型**：

| 选项 | 方案 | 说明 |
|------|------|------|
| 图表渲染 | 纯 SVG + CSS | 轻量，无额外依赖，支持动画 |
| 动画 | CSS transitions + keyframes | Agent 状态变化过渡动画 |
| 连线 | SVG path 贝塞尔曲线 | 节点间箭头连线 |
| 布局 | 手动计算（拓扑排序） | 7 个节点，固定布局 |
| 状态管理 | React useReducer | 维护 agents 状态数组 |

**对比 React Flow**：不采用 React Flow 的原因是管线拓扑固定（线性 DAG），无需拖拽编辑，纯 SVG 方案更轻量且可控。
