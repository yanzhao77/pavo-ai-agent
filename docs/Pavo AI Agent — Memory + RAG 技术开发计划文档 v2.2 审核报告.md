4.2 技术方案评估
技术决策	评估	说明
纯 SVG + CSS	✅ 优秀	7 个节点线性布局，无需引入 React Flow，减少依赖和维护成本
手动拓扑计算	✅ 合理	拓扑固定（线性 DAG），无需动态布局算法
useReducer + context	✅ 合理	状态管理清晰，适合 SSE 事件驱动的更新模式
SSE 向后兼容	✅ 重要	新增字段不影响旧版 ChatPanel
对比分析：

方案	优点	缺点	适用场景
纯 SVG（当前方案）	轻量、可控、无额外依赖	需手写布局代码	固定拓扑、节点数少（7个）
React Flow	功能强大、拖拽编辑	包体积大、学习成本	复杂 DAG、需要交互编辑
ECharts/D3	图表丰富	过重	数据可视化为主
结论：当前技术选型合理，适合 v2.2 的需求（展示为主，非编辑）。
