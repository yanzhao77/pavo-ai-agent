## 1. 项目概述

### 1.1 背景

Pavo AI Agent 当前是一个传统的 Web 应用架构：

- **前端**：Next.js 14 → HTTP/SSE → **后端 FastAPI** → **Agnes AI 网关**
- **后端**：FastAPI + PostgreSQL + Redis + MinIO
- **AI 智能体**：7 个 Agent 内嵌于后端进程中

此架构要求用户必须通过浏览器或直接调用 REST API 才能与系统交互。缺少一个"AI 原生"的接口层——让 AI 编程工具（如 Cursor、Claude Code、Codex）和 AI 应用能够以标准化协议直接发现、调用系统的能力。

### 1.2 目标

在现有 Pavo AI Agent 基础上新增一个 **MCP Server 层**，实现：

| 目标 | 说明 |
|------|------|
| 标准化接口 | 通过 MCP 协议暴露 Pavo 的核心能力，让任意 MCP 客户端可发现和调用 |
| AI 原生交互 | AI 编程工具可以直接调用项目创建、分镜生成、视频渲染等操作 |
| 资源化数据 | 项目、角色、分镜等数据以 MCP Resource 形式暴露，支持自动补全和上下文注入 |
| 提示词模板 | 将 Pavo 的 Agent Prompt 工程经验封装为 MCP Prompt 模板 |
| 非侵入接入 | 不改变现有 FastAPI 架构，MCP Server 作为独立进程运行 |

### 1.3 范围

| 包含 | 不包含 |
|------|--------|
| MCP Server 独立进程（Python） | 修改现有 FastAPI 路由 |
| MCP Tools：8 个核心工具 | 替换现有前端 |
| MCP Resources：7 个资源模板 | 修改数据模型 |
| MCP Prompts：3 个提示词模板 | 修改 Agent 管线 |
| 认证与鉴权适配 | 删除现有功能 |
| Docker Compose 集成 | |
| CI 测试集成 | |

## 2. MCP 协议简介

### 2.1 什么是 MCP

MCP (Model Context Protocol) 是由 Anthropic 主导的开放协议，定义了 AI 模型与外部工具/数据源之间的标准化交互接口。类比于 USB-C 为硬件外设提供的统一接口，MCP 为 AI 应用提供了统一的工具集成标准。

### 2.2 核心概念

| 概念 | 说明 | 类比 REST |
|------|------|-----------|
| **Tools** | AI 模型可以调用的函数，执行动作并返回结果 | POST /api/action |
| **Resources** | AI 模型可以读取的结构化数据资源 | GET /api/resource/{id} |
| **Resource Templates** | 带参数的资源 URI 模板 | GET /api/resources/{type}/{id} |
| **Prompts** | 可复用的提示词模板 | 类似 OpenAPI 的示例请求 |
| **Transport** | 通信协议：stdio（子进程）或 SSE（HTTP） | HTTP |

### 2.3 MCP 的适用场景

对于 Pavo AI Agent 项目，MCP Server 层主要服务于以下场景：

| 场景 | 描述 | 优先级 |
|------|------|--------|
| **AI IDE 集成** | 开发者在 Cursor/VS Code 中通过 MCP 直接创建项目、生成分镜 | P0 |
| **自动化流水线** | CI/CD 中通过 MCP 工具链自动生成视频内容 | P1 |
| **智能体协作** | 其他 AI Agent 系统通过 MCP 调用 Pavo 的分镜能力 | P1 |
| **数据探查** | AI 编程工具直接读取 Pavo 数据库中的项目数据 | P2 |

### 3.2 MCP Server 进程模型

MCP Server 作为**独立 Python 进程**运行，通过以下方式与现有系统交互：

```
┌──────────────────────────────────────────────┐
│           Pavo MCP Server (mcp_server/)       │
│                                              │
│  ┌──────────────┐  STDIO or SSE Transport    │
│  │  MCP SDK     │ ◄───────────────────────  │
│  │  (mcp lib)   │     MCP Client             │
│  └──────┬───────┘                            │
│         │                                    │
│  ┌──────┴──────────────────────────────┐     │
│  │   Service Adapter Layer             │     │
│  │   - 复用 app/services/* 的类/函数    │     │
│  │   - 通过数据库 Session 直接读取数据   │     │
│  │   - 调用 ProjectService 执行工作流   │     │
│  └─────────────────────────────────────┘     │
└──────────────────────────────────────────────┘
```

**两种 Transport 模式**：

| 模式 | 适用场景 | 配置方式 |
|------|---------|---------|
| **stdio** | VS Code/Cursor 等本地 IDE 集成 | 默认模式，通过子进程通信 |
| **SSE** | 远程服务器部署、跨网络调用 | 通过 HTTP 端点通信 |

### 3.3 与现有架构的关系

| 维度 | 策略 |
|------|------|
| **代码复用** | MCP Server 直接复用 `app/services/` 和 `app/models/` 中的代码 |
| **数据库** | 直接连接同一 PostgreSQL 数据库（只读+写入） |
| **认证** | MCP 工具调用需要传入 Token，由 `auth_adapter.py` 校验 |
| **部署** | Docker Compose 中增加 `mcp-server` 服务 |
| **配置** | 复用 `.env` 中的 `DATABASE_URL`、`AGNES_API_KEY` 等配置 |

## 5. MCP Resources 详细设计

### 5.1 总览

| Resource URI | 说明 | 返回格式 | 优先级 |
|-------------|------|---------|--------|
| `pavo://projects` | 项目列表 | 项目摘要数组 | P1 |
| `pavo://projects/{id}` | 项目详情 | 完整 Project JSON | P0 |
| `pavo://projects/{id}/storyboard` | 分镜数据 | Storyboard JSON | P0 |
| `pavo://projects/{id}/characters` | 角色列表 | Character JSON 数组 | P0 |
| `pavo://projects/{id}/scenes` | 场景列表 | Scene JSON 数组 | P1 |
| `pavo://projects/{id}/props` | 道具列表 | Prop JSON 数组 | P1 |
| `pavo://projects/{id}/trace` | Agent 执行轨迹 | Trace Log 数组 | P2 |

### 5.2 Resource 实现模式

```python
# Resource 模板注册
@mcp.resource("pavo://projects/{project_id}")
async def get_project_resource(project_id: str) -> str:
    """获取指定项目的完整数据。"""
    project = await project_adapter.get_project(project_id)
    return json.dumps(project, ensure_ascii=False, indent=2)

# Resource 列表
@mcp.resource("pavo://projects")
async def list_projects_resource() -> str:
    """获取所有项目列表。"""
    projects = await project_adapter.list_projects()
    return json.dumps(projects, ensure_ascii=False, indent=2)
```

### 5.3 Resource URI 设计规范

| 规范 | 说明 |
|------|------|
| Scheme | `pavo://` |
| 命名空间 | `projects`、`projects/{id}/{sub}` |
| 子资源 | `storyboard`、`characters`、`scenes`、`props`、`trace` |
| 序列化 | JSON，UTF-8，缩进 2 空格 |
| MIME Type | `application/json` |

### 6.2 Prompt 设计

#### Prompt: generate_storyboard

```python
@mcp.prompt()
def generate_storyboard(input: str, characters: str = "", scenes: str = "") -> list[Message]:
    """
    根据故事输入、角色和场景信息，生成完整的分镜提示词。

    适用于需要 AI 辅助生成视频分镜的场景。
    """
    return [
            SystemMessage(content="""你是一个专业的视频分镜师和编剧。
    根据用户的故事描述，创建一个逐镜头的视频分镜脚本。

    输出要求：
    1. 按幕次组织，每幕包含标题、总时长、关键帧画面
    2. 每个镜头包含：编号、景别、运镜、角度、画面描述、对白、时长
    3. 遵循起承转合叙事结构，总时长 30-60 秒
    4. 输出必须是有效的 JSON 格式
    """),
            UserMessage(content=f"""
    故事：{input}
    角色：{characters}
    场景：{scenes}
    请生成完整分镜。""")
    ]
```

#### Prompt: design_characters

```python
@mcp.prompt()
def design_characters(story: str, genre: str = "") -> list[Message]:
    """
    根据故事描述设计 2-4 个角色档案。
    """
    return [
        SystemMessage(content="设计角色的名称、年龄、外貌、性格、声音和人物关系。"),
        UserMessage(content=f"故事：{story}\n类型：{genre}")
```

## 7. 数据结构与序列化

### 7.1 MCP 专用 Schema

MCP Server 需要定义轻量级的序列化 Schema，用于工具/资源的输入输出。

```python
# mcp_server/models/mcp_schemas.py

from pydantic import BaseModel
from typing import Optional


class MCPProjectSummary(BaseModel):
    """项目摘要（列表用）"""
    id: str
    title: str
    status: str
    created_at: str


class MCPToolResult(BaseModel):
    """工具调用标准返回"""
    success: bool
    data: dict | list | None = None
    error: str | None = None


class MCPCreateProjectInput(BaseModel):
    """创建项目输入"""
    input: str
    user_id: str = ""


class MCPRegenerateInput(BaseModel):
    """重生成模块输入"""
    project_id: str
    module: str  # storyboard | characters | scenes
```

### 7.2 序列化规则

| 场景 | 规则 |
|------|------|
| Tool 输出 | 统一 `MCPToolResult` 包裹，`success` + `data`/`error` |
| Resource 输出 | 直接输出 JSON 字符串，`ensure_ascii=False` |
| 日期格式 | ISO 8601 字符串 |
| UUID 格式 | 字符串格式 |
| 长文本 | 截断至 10000 字符，附带 `_truncated` 标记 |

### 13.2 最小 Server 示例

```python
from mcp.server import Server
import mcp.server.stdio

# 创建 Server 实例
server = Server("pavo-mcp-server")

@server.list_tools()
async def list_tools():
    return [
        {
            "name": "hello",
            "description": "A simple hello world tool",
            "inputSchema": {
                "type": "object",
                "properties": {
                    "name": {"type": "string"}
                }
            }
        }
    ]

@server.call_tool()
async def call_tool(name: str, arguments: dict):
    if name == "hello":
        return f"Hello, {arguments['name']}!"

async def main():
    async with mcp.server.stdio.stdio_server() as (read, write):
        await server.run(read, write)

if __name__ == "__main__":
    import asyncio
    asyncio.run(main())
```
