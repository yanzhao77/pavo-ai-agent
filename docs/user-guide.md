## 1. 产品简介

Pavo AI Agent 是一个 AI 驱动的视频分镜生成平台。用户只需用自然语言描述一个故事创意，系统会自动调用 7 个专用 AI 智能体协作生成：

- **角色设定** — 姓名、年龄、外貌、性格、声音
- **场景设计** — 环境、灯光、氛围、时段
- **道具编目** — 核心道具的外观、交互方式、叙事意义
- **分镜脚本** — 逐场逐镜的景别、运镜、画面描述、对白、时长
- **视频生成** — 将分镜转化为 AI 视频（可选）

### 适用场景

- 短视频内容创作者（抖音、B站、视频号）
- 广告创意与脚本策划
- 影视前期分镜设计
- 故事板可视化教学
- 个人创意表达

## 2. 快速上手

### 2.1 环境准备

确保系统已启动（参见项目 README 的"快速开始"章节）。

### 2.2 首次使用

**步骤 1：登录系统**

打开浏览器访问 `http://localhost:3000`，输入一个用户名（任意字符）即可登录。系统会为你生成一个认证 Token，保存在浏览器本地。

**步骤 2：输入故事**

在左侧"对话面板"的输入框中，用自然语言描述你的故事创意。例如：

> *一位父亲下班疲惫地回到家，他 5 岁的儿子端来一盆热水，认真地为爸爸洗脚。*

点击发送按钮（或按 Enter）。

**步骤 3：观察智能体工作**

发送后，对话面板会显示 7 个智能体的实时工作进度：

```
[Planner] 正在分析故事... ✓
[Character] 正在生成角色... ✓
[Scene] 正在生成场景... ✓
[Prop] 正在生成道具... ✓
[Storyboard] 正在生成分镜... ✓
[Reviewer] 正在审查质量... ✓
[Fixer] 正在修正问题... ✓
```

每个步骤完成后，右侧预览面板会同步更新。

**步骤 4：查看结果**

生成完成后（约 10-30 秒），右侧预览面板会显示完整的分镜脚本，包含多个标签页：

- **分镜** — 逐场逐镜的完整视图
- **角色** — 生成的 2-4 个角色档案
- **场景** — 2-4 个场景设计
- **道具** — 核心道具列表

### 2.3 一句话示例

试试这些故事创意：

| 故事类型 | 示例输入 |
|---------|---------|
| 亲情 | 一位父亲下班疲惫地回到家，他5岁的儿子端来一盆热水，认真地为爸爸洗脚 |
| 悬疑 | 午夜图书馆，一本旧书自动翻开，书中走出一个影子 |
| 治愈 | 雨天的咖啡馆，一只流浪猫推门而入，坐到了唯一空着的座位上 |
| 励志 | 清晨五点，一位老人在空无一人的体育馆里独自投篮 |

```
|  ✓ Scene         |  │ 父亲推开门...        │       |
|  ...             |  ├──────────────────────┤       |
|                  |  │ 镜头 2: 近景 | 推近  │       |
|                  |  │ 儿子端水盆...        │       |
|                  |  └──────────────────────┘       |

```

### 3.2 对话面板（左侧）

- **输入框** — 输入故事创意，按 Enter 发送
- **智能体日志** — 实时显示每个 Agent 的执行状态
- **生成中状态** — 加载动画提示用户等待

### 3.3 预览面板（右侧）

预览面板包含四个标签页：

**分镜标签页**
- 按场景（Act）组织，每个场景包含多个镜头
- 每个镜头显示：镜头编号、景别、运镜、角度、画面描述、对白、时长、出场角色
- 支持拖拽排序（拖动镜头卡片调整顺序）

**角色标签页**
- 每个角色以卡片形式展示
- 包含：姓名、角色类型、年龄、外貌描述、性格标签、声音描述、关系

**场景标签页**
- 每个场景以卡片形式展示
- 包含：场景名称、时段、环境描述、灯光描述、氛围

**道具标签页**
- 每个道具以卡片形式展示
- 包含：名称、类型、外观、交互方式、叙事意义

## 4. 核心功能详解

### 4.1 故事输入

系统接受任意长度的自然语言描述。好的输入通常包含：

- **核心人物** — 谁在故事中？
- **核心事件** — 发生了什么？
- **场景暗示** — 发生在哪里？
- **情感基调** — 温暖、悬疑、感人？

### 4.2 智能体管线

系统内部由 7 个专用 AI 智能体协作完成：

| 阶段 | 智能体 | 产出 | 说明 |
|------|--------|------|------|
| 1 | 规划师 | 故事分析 | 提取主题、情感、核心元素、预估时长 |
| 2 | 角色智能体 | 角色档案 | 基于故事分析生成 2-4 个角色 |
| 3 | 场景智能体 | 场景设计 | 基于角色和故事生成 2-4 个场景 |
| 4 | 道具智能体 | 道具清单 | 识别并设计核心道具 |
| 5 | 分镜智能体 | 逐镜分镜 | 生成完整的逐镜头脚本 |
| 6 | 审查智能体 | 质量报告 | 检查一致性和完整性 |
| 7 | 修复智能体 | 修正版本 | 自动修复审查发现的问题 |

### 4.3 分镜结构

分镜遵循中国古典叙事结构 **起承转合**：

- **3-4 幕** — 自然递进
- **总时长 30-60 秒** — 适合短视频
- **每幕含 BGM 和 SFX 描述**
- **每个镜头指定**：
  - **景别**：远景 / 全景 / 中景 / 中近景 / 近景 / 特写
  - **运镜**：固定 / 横移 / 推近 / 拉远 / 摇移 / 跟拍
  - **角度**：平视 / 平视微俯 / 平视侧面
  - 画面描述、对白、时长、出场角色

### 4.4 导出功能

系统支持三种导出格式：

- **Markdown** — 适合文档归档和分享
- **纯文本** — 适合复制到其他工具
- **PDF** — 适合打印和正式交付

导出路径：在分镜预览页面点击导出按钮即可。

## 5. 高级功能

### 5.1 重新生成模块

如果对某个模块（如角色、场景、分镜）不满意，可以单独重新生成：

```bash
POST /api/projects/{projectId}/regenerate
{
  "module": "characters"  // 可选: characters, scenes, storyboard
}
```

### 5.2 版本历史

每次重要修改会自动创建版本快照，你可以：

- **查看版本列表** — 浏览所有历史版本
- **回滚到任意版本** — 一键恢复

### 5.3 视频生成

在分镜生成完成后，可以触发视频生成：

1. 点击"生成视频"按钮
2. 系统会为每个镜头构造视频提示词
3. 调用 Agnes AI 视频模型逐镜生成
4. 生成的视频存储在 MinIO 中

> **注意**：视频生成需要 API Key 有视频模型权限，且生成时间较长。

### 5.4 手动编辑

预览面板支持直接编辑：

- 拖拽镜头卡片调整顺序
- 后续将支持直接编辑角色、场景字段

## 6. 常见问题

### Q: 生成的分镜不够详细怎么办？

A: 输入更详细的故事描述，包含人物外貌、场景细节、情感变化等。也可以使用"重新生成"功能对特定模块重新生成。

### Q: 角色一致性如何保证？

A: 每个角色都有 `consistencyKey` 字段（外貌关键特征摘要），在后续所有镜头中作为 Prompt 前缀自动传入。

### Q: 可以生成多长时间的脚本？

A: 默认总时长控制在 30-60 秒。如需更长，可以在故事输入中指定，如"这是一个 3 分钟的故事"。

### Q: 视频生成失败怎么办？

A: 检查 API Key 是否有视频模型权限，确认 MinIO 服务正常运行。如仍失败，可以尝试单镜重新生成。

### Q: 如何分享我的项目？

A: 使用"导出"功能导出为 Markdown 或 PDF 进行分享。

## 7. 最佳实践

### 7.1 输入技巧

- **提供具体细节**："一个穿红色连衣裙的小女孩" 优于 "一个小女孩"
- **包含情感线索**："疲惫但温暖" 比仅描述动作更能引导 AI
- **设定场景**："雨天的公交站" 比 "一个地方" 更丰富
- **指定长度** 如需要特定时长，在描述中包含

### 7.2 迭代工作流

1. **初稿生成** — 输入核心故事，获取第一版分镜
2. **审查修改** — 审查生成结果，记下需要调整的地方
3. **模块重生成** — 对不满意的模块单独重生成
4. **导出交付** — 满意后导出为 Markdown 或 PDF

### 7.3 使用建议

- 先用简单的故事测试系统（1-2 个角色，1-2 个场景）
- 熟悉后再尝试复杂的故事
- 利用版本历史功能大胆尝试不同方向
- 使用导出功能建立自己的分镜素材库
