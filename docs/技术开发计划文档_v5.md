## 开发原则

### P1. Phase-by-Phase 阶段开发
严禁跳阶段开发。每个 Phase 必须完成交付物并通过测试验证后方可进入下一个 Phase。

### P2. 后端优先
API 和数据库更改必须在前端开发前完成并验证。

### P3. 层级验收
单元测试 → 集成测试 → E2E 测试，三级验证缺一不可。
